2. Adding YouTube links to recipe pages
While exploring the response data returned by TheMealDB API, I noticed that each recipe contained a YouTube tutorial link, which I wanted to add it into my page as well,
So I asked Claude "How can I add the YouTube link returned by the API to my recipe page?"
After also copy paste an TheMealDB result JSON page, Claude explained that the API response already contained a strYoutube field and suggested passing this value from Flask into the recipe template and conditionally rendering a button or hyperlink.
Based on this explanation, I added the following functionality:
Retrieved strYoutube from the recipe data
Passed the value to recipe.html
Added a "Watch on YouTube" button that appears only when a video link is available

3. Debugging a Flask runtime error
While testing the application, I received the following error:
AttributeError: 'NoneType' object has no attribute 'strip', which I don't understand what it means after checking my code line by line,
So I copied the entire terminal error message into Claude and asked: "What does this error mean?"
Claude explained that .strip() can only be called on strings, and that one or more ingredient fields returned by the API were likely None rather than text.
It explained that the code was effectively trying to execute:
None.strip()
which caused the application to crash.
After receiving this explanation, I modified the ingredient extraction logic to safely handle missing values before calling .strip().

5. Finding the reason for slow performance
Even after the website was functioning correctly, I noticed that recipe searches were still relatively slow.
I asked Claude:"Why does the recipe search take so long after users enter ingredients?"
Claude explained that the application was performing many API requests sequentially:
Search recipes from TheMealDB
Request details for each recipe
Request calorie information from USDA for multiple ingredients
Because each request waits for the previous request to finish, the total response time becomes noticeable.
I then asked:
"How can I make it faster?"
It give me specific instructions on reducing duplicate API requests, limiting the number of recipes processed, and making requests concurrently
Although I understood the general explanation, I did not fully understand how to implement some of the more advanced optimization techniques.
And the waiting time is not that unacceptable long, therefore, I decided not to make further modifications in this area and kept the existing implementation.
